文章结构：

# {主题名称}

## 一、先说结论

用 1 到 3 句话说清楚这个概念的本质。

要求：

- 不要绕。
- 不要先铺垫历史。
- 直接给出我应该记住的核心判断。

## 二、为什么需要理解它

说明这个概念为什么重要。

可以从这些角度选择：

- 它解决了什么问题；
- 如果不理解它，容易产生什么误解；
- 它和 AI 应用开发有什么关系；
- 它在 RAG / Prompt / Agent / LLM 调用中有什么作用。

## 三、举一个直观例子

用一个简单例子解释这个概念。

要求：

- 例子要具体；
- 最好能和编程、前端、函数、接口、数据库或日常场景类比；
- 不要只给定义，要让人有画面感。

## 四、本质是什么

从更底层解释它到底是什么。

要求：

- 如果能用公式或一句话总结，可以给出来；
- 如果涉及流程，请按步骤拆解；
- 如果适合对比，请用表格。

## 五、和相近概念的区别

例如：

- LLM vs 传统程序；
- Token vs 字 / 词；
- Prompt vs Prompt Engineering；
- RAG vs Fine-tuning；
- Agent vs 普通聊天。

如果没有明显相近概念，可以写“常见理解误区”。

## 六、对工程实践的影响

说明理解这个概念后，写代码或设计 AI 系统时会有什么变化。

可以包含：

- 对 Prompt 设计的影响；
- 对 RAG 检索的影响；
- 对模型选型的影响；
- 对成本、稳定性、延迟、幻觉治理的影响；
- 在我的 RagFlow 项目里可能对应哪个模块。
