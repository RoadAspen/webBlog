# Blog

::: tip 关于这个模块
这里主要记录前端开发中的知识点、原理理解和工程实践，重点放在 JavaScript、浏览器、网络、工程化与相关生态。
:::

## 这个模块写什么

`blog` 目录更像一个前端知识仓，而不是单纯的随笔区，内容主要覆盖：

- JavaScript 基础与语言机制
- HTTP、浏览器与网络通信
- 前端工程化与构建工具
- TypeScript、Node.js 与服务端基础
- 常见框架、状态管理与生态对比
- 性能优化、安全、微前端等专题

## 目录概览

- `js`：JavaScript 基础、运行机制、原型、闭包、Promise、事件循环等
- `http`：HTTP 协议、缓存、SSE、WebSocket、网络基础
- `css`：布局、BFC、单位、伪类伪元素等样式基础
- `ts`：TypeScript 类型系统与常见概念
- `node`：Node.js 与 Koa 等后端基础
- `engineering`：前端工程化、规范、Monorepo、工具链
- `webpack`：Webpack 原理与构建相关内容
- `framework`：常见前端框架与状态管理方案
- `libraryDiff`：常见能力选型与方案对比
- `optimization`：前端性能优化与浏览器加载流程
- `operation`：部署、运维相关内容
- `safety`：前端监控与安全相关内容
- `microfrontends`：微前端相关记录
- `nestjs`：NestJS 实践
- `reactnative`：React Native 相关内容
- `设计模式`：设计模式与抽象思路

## 适合怎么读

如果是按学习路径来读，比较建议这样看：

1. 先看 `js`、`css`、`http`
2. 再看 `ts`、`node`、`framework`
3. 然后进入 `engineering`、`webpack`、`optimization`
4. 最后按专题补充 `safety`、`microfrontends`、`nestjs` 等内容

## 为什么要写这些

写这些内容，主要是为了把零散知识沉淀成可复用的理解：

1. 记录学习和实践过程
2. 梳理前端知识体系，而不只是记结论
3. 沉淀常见问题、原理和工程经验
4. 让之后回顾、补漏、扩展都更轻松

后续会继续把旧文章整理得更清楚，也会逐步补上更成体系的索引。
