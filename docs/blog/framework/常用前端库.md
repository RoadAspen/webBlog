## 前言

这里罗列了很多 React 生态相关的前端库，主要分为 UI 库、CSS 预处理、CSS in JS、React 状态管理、表单库、hooks 封装等，主要用作 To B SAAS 平台搭建。

## UI 库

> UI 库目前分为 全量型和 headless(及原子化)，各有优缺点，全量型的代表就是 Antd，headless 的代表就是 Material UI。

**不推荐从零开发自己的 UI 库**

1. **时间和成本：** 重新开发 UI 组件库需要投入大量时间和资源。使用现成的 UI 组件库可以满足需求，并且定制化成本较低。
2. **维护和更新：** 一个成熟的 UI 组件库通常由一个活跃的社区或团队维护和更新。自己重新开发的组件库可能需要更多的精力来持续维护和更新，尤其是在长期使用过程中需要适应新的技术和需求。
3. **稳定性和兼容性：** 市面上现有流行的组件库都是经过了大量的测试和优化，可一提供相对稳定的性能和跨浏览器的兼容性。重新开发的组件可能会面临更多的稳定性和兼容性问题。
4. **功能和特性：** 现有的 UI 组件库通常经过长时间的迭代和优化，拥有非常丰富的功能和特性，并且提供了丰富的 API 和文档，可以满足企业级 Web 应用的大部分场景。
5. **学习曲线：** 使用流行的 UI 组件库，一般具有良好的 playground，可以快速上手，较少学习曲线，同时在人才市场上容易找到熟悉相同组件库的开发人员，

### Antd

Ant Design 是一套优秀的企业级 UI 设计语言和 React 组件库，由阿里巴巴 Ant Design 设计团队开发。Ant Design 包括了一系列高质量的 React UI 组件，以及一套设计规范和风格指南，能够帮助开发者快速构建出美观、易用的企业级 Web 应用。

### 优点

1. 丰富的组件库： Antd 提供了丰富的 UI 组件，覆盖了常见的企业级应用场景，包括按钮、表单、表格、导航、布局等，可以帮助开发者快速构建出美观、功能丰富的应用界面。

2. 设计规范和风格指南： Antd 提供了一套完整的设计规范和风格指南，包括设计原则、视觉规范、交互规范等，可以帮助开发者保持一致的设计风格和用户体验。

3. 基于 React 的组件库： Antd 是基于 React 的组件库，采用了现代化的前端技术栈，具有优秀的性能和开发体验，可以与 React 生态系统无缝集成。

4. 模块化和可定制化： Antd 的组件采用了模块化的设计，可以灵活地组合和定制各种组件，满足不同项目的需求。同时，Antd 提供了丰富的主题定制功能，可以根据项目需求定制自己的主题风格。

5. 良好的文档和社区支持： Antd 提供了详细的文档和示例，帮助开发者快速上手和使用组件。同时，Antd 拥有一个活跃的开发者社区，提供了丰富的资源和支持。

### 缺点

1. 体积较大， antd 提供了非常丰富的 UI 组件，覆盖了常见的企业应用场景，因此它的打包体积较大，后期比较依赖于性能优化。
2. 学习曲线较陡，组件丰富，有很多的组件，学习成本较大，需要对组件进行深究
3. 不适合个性化项目，如果项目追求个性化的样式及交互逻辑，可能需要覆写很多已有样式

## 原子 css

> 原子 css 是将单个的 css 属性整合成一个语义化的类名，我们通过给组件添加对应属性的类名，来快速的组合出我们想要的 css。目前社区比较活跃的有 Tailwind、unoCss。

### 采用原子 css 的优缺点

1. 原子 css 的类名命名规范简单，让我们不用去思考变量名，自由排列组合，极大的节省了写样式的时间。
2. 占用空间相较于传统的打包方式的 css 文件占比更小。
3. 初始接触原子 css 时需要花费很多时间去查阅文档。
4. 当一个元素的样式比较复杂时，需要编写大量样式，而原子 css 可以减少很多工作量，但这也造成了 className 变得巨长无比，可读性变差。

### [Tailwind CSS](https://tailwindcss.com/docs/installation)

> Tailwind CSS 是一个实用的 CSS 框架，它提供了一组预定义的原子类，用于快速构建界面。与传统的 CSS 框架不同，Tailwind CSS 不是基于组件的，而是基于原子类的，这意味着开发者可以直接在 HTML 中使用各种原子类来定义样式，而不需要编写自定义的 CSS。

#### 优点

1. 预定义样式充足，可以快速的构建界面。
2. 支持自定义主题，提供了丰富的原子类和工具类。
3. 社区庞大，有丰富的生态系统，提供了大量的文档、示例、和扩展库

#### 缺点

1. 因为采用了预定义样式，生成 css 文件较大（文件变大程度一般可以忽略）。

### unoCss

UNO CSS 也是一种基于原子类的 CSS 框架，采用 JIT 技术，根据项目中实际使用的样式动态生成 CSS，以减少生成的 CSS 文件大小。

#### 优点

1. UNO CSS 使用 JIT 技术，根据项目中实际使用的样式动态生成 CSS，不会包含未使用的样式，因此生成的 CSS 文件大小通常较小，可以提高网页加载速度和性能。
2. 支持自定义主题，提供了可扩展的原子类和工具类。

#### 缺点

1. 社区相对较小，在发展壮大中。

## storage 操作

用于简化 storage 操作，突破 浏览器 localStorage 的存储大小限制，在可支持的环境中，优先使用 indexDB 作为本地存储方案。

### localStorage

使用浏览器原生自带的 localStorage 和 sessionStorage 作为前端缓存的本地缓存方案。

#### 优点

1. 可以立即使用，兼容性良好，兼容所有浏览器

#### 缺点

1. 有大小限制

### localForage

localForage 是一个 JavaScript API，用于在浏览器中使用 localStorage、WebSQL、IndexedDB 或者 FileSystemStorage。它提供了一个统一的接口来存储和获取数据，而不需要考虑数据存储的细节。

#### 优点

1. 兼容性良好，兼容所有浏览器
2. 使用 IndexDB,可以存储大量数据
