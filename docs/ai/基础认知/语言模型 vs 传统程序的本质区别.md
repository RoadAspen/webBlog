# 语言模型 vs 传统程序的本质区别

## 为什么一个写代码的人，需要重新理解“程序是什么”

## 一、先说结论

语言模型（LLM）与传统程序的根本区别：

```sh
传统程序 = 人写的规则，机器执行
语言模型 = 机器学习规律， 按规律生成
```

说的再直白一点：

- 传统程序： 你决定怎么做
- 语言模型：你只描述目标，它自己“计算”怎么做

## 二、举一个例子

> 任务：判断一句话是“正面”还是“负面”

**传统程序写法**：

```JS
if(text.includes("好")) return "正面"

if(text.includes("差")) return "负面"
```

**问题：**

- 覆盖不全
- 表达方式很多
- 很难维护

**语言模型写法**：

```sj
人： 请判断这句话是正面还是负面： “这个产品真的让我很惊喜”

模型输出：正面
```

**本质的区别：**

> 传统程序： 你写规则
>
> 语言模型： 规则在模型里（训练出来）

## 三、本质差异一： 确定性 vs 概率性

**传统程序**：输入一样 → 输出永远一样

```JS
function add(a,b) {return a+b}

add(2,2)  // 4
```

如果你这样写

```JS
function random(){
    return Math.random()
}
raddom() // 0-1之间的任意随机数
```

本质上，无论运行多少次，依然符合你生成一个随机数的期望

**语言模型**：输入一样 → 输出可能不同

因为：

- 回答是概率分布生成
- 不是固定的规则

再比如：

> 输入： 今天天气很好

模型的输出可能是：

```sh
回答一  心情不错 （倾向于心态）
回答二 适合出去走一走 （倾向于旅游）
```

模型的回答就是这样设计的

## 四、本质差距二：规则驱动 vs 数据驱动

传统程序：**逻辑 = 人写的代码**

语言模型： **逻辑 = 从数据中学习来的模式**

比如你从来没有说**惊喜是正面**，但是模型知道，因为模型的训练包中包含了很多把惊喜等同于正面的数据

## 五、本质差距三： 精确理解 vs 模糊理解

**传统程序： 必须精确匹配**

```JS
 if（text == “你好”） return 你好
```

**语言模型： 可以理解为“类似表达”**

> 你好、您好、哈喽 都是打招呼的意思

本质是：

> 传统程序： 符号匹配
>
> LLM： 语义理解

## 六、本质差距四： 一步计算 vs 逐词生成

传统程序：

```JS
一次执行
    ↓
得到结果
```

语言模型： 一个 token 一个 token 逐步生成（token by token）

```JS
实际的过程
第 1 步： 生成一个词
        ↓
第 2 步：基于上下文再生成一个词
        ↓
第 3 步：基于上下文再生成一个词
...
```

总结为 自回归生成

## 七、本质差异五：工具 vs 合作对象

**举个例子：写代码**

- 传统程序： 你控制它，每个字母都由你决定
- 语言模型： 你引导它，实际结果受你的语言引导，但是不能完全控制

本质的变化：

> 从写代码 → 写提示（Prompt）

## 八、一个容易产生的误区

**错误理解： LLM = 更聪明的程序**

**正确理解： LLM = 一种全新的计算范式**，LLM 的输出结果是否符合你的期望，在于你的引导以及它的学习资料

它不是：

- 更强的 if-else
- 更高级的函数

而是：

> 基于概率的语言生成系统

## 九、对程序设计的影响

**如果使用传统思维设计 AI 系统：**

- 想写规则
- 想控制细节
- 想让它 100%正确

这几点都是做不到的

**如果使用 LLM 思维：**

- 设计输入解析（Prompt）
- 控制随机概率（Temperature / Top-p）
- 接受一丝的不确定性（99.99999999%）
- 做结果校验（对那一点的不确定性做好拦截）

## 十、总结

通过上边的对比，我们需要达成一个共识：

> 传统程序： 执行确定的逻辑
>
> 语言模型： 预测下一个可能的词
