# 为什么“语义相似”可以被计算

上一章我们讲了, Embedding 是如何将文字转换成向量，并把文本映射到一个高维的语义空间。。
在这个空间中

- 相似的语义内容距离更近。
- 不同的语义内容距离更远。

那么, 一个新的问题:

> 为什么一串数字之间的距离，可以代表两个文本的语义相似程度？

## 一、先说结论

语义相似度之所以可以被计算

> **因为 Embedding 模型在训练过程中，学习了大量文本之间的关系,并将这种关系编码到了向量空间中**

简单来说：

> 模型不是人为规定“苹果”和“香蕉”应该靠近，而是在大量文本中学习出了它们之间的关系。

## 我们需要达成一个共识

市面上可用的 Embedding 模型也是经过大量的数据训练出来的,所以它能学习到语义关系。

1. 如果 Embedding 模型没有经过任何的训练,那么它就无法表达语义关系。输出的就是是一个完全随机的向量,而不是有意义的语义向量。

2. 在 Embedding 训练的开始,模型会对一些文字进行随机初始化,当训练的过程中,它会根据输入的文本的关联度,不断的调整这些向量,直到这些向量能够准确表达语义关系为止。

3. 当我们使用一个训练好的 Embedding 模型,我们得到的向量是具有语义含义的。

接下来我们说的都是基于训练好的 Embedding 模型。

## 二、Embedding 并不是简单转换文字

很多人认为: 苹果 -> [0.12, -0.35, 0.67, ...]

- 第一个数字代表水果
- 第二个代表颜色

不是的!

Embedding 并不是给每个词打标签。
它做的是:

> 根据大量的文本中的使用规律,学习每个词,每句话在语义空间中的位置

## 三、模型是如何学习语义关系的？

假设模型在训练过程中看到大量这样的文本：

- 我喜欢吃苹果
- 我喜欢吃香蕉
- 我喜欢吃西瓜

模型发现:有一些词经常出现在类似的位置:

> 喜欢吃\_\_

于是模型就逐渐学习:

苹果、香蕉、西瓜 它们在同一个语境下出现，所以它们是具有某种相似的语义关系的。

**因此: 它们在语义空间中会彼此靠近。**

再例如：
模型看到：

- 苹果发布了新手机
- 苹果推出了新的电脑

这里的"苹果"和：

手机
电脑
芯片
科技公司

一起出现。

所以模型会学习到：

这里的苹果代表的是：Apple 公司,而不是水果。

## 四、语义空间是如何形成的？

注意: **多维空间无法可视化,但是我们可以通过降维来观察它。**

可以把 Embedding 训练过程理解成：不断的调整地图上的位置

```
苹果       ●

香蕉                 ●

手机                       ●

```

模型不知道它们之间的关系。随着不断的训练,模型发现, 苹果和香蕉和水果经常出现在相似的环境,于是调整一下:

```
苹果       ●

香蕉        ●

水果          ●

```

同时: 手机,电脑, 芯片 和科技公司经常出现在相似的环境,于是调整一下:

```
          水果区域
       苹果 ●
            ● 香蕉
                         科技区域

                      手机 ●
                         ●电脑

```

这样就是一个语义空间的形成过程。

## 五、为什么距离代表了语义相似度？

从语义空间的形成过程可以看出,在这个空间中：刚开始位置是随机的, 随着训练的进行,这些向量的位置逐渐变得有序。经过模型训练，它的位置被赋予了语义关系。

## 六、如何计算两个向量的相似度

在模型中使用两个向量的夹角余弦值来衡量它们的相似度。

```
[a1, a2, a3, ... , an] // 向量 1

[b1, b2, b3, ... , bn] // 向量 2

// 计算两个向量的夹角余弦值公式
相似度  = cosα = (A·B)/(|A|*|B|) = (a1*b1 + a2*b2 + ... + an*bn) / (sqrt(a1^2+a2^2+...+an^2) * sqrt(b1^2+b2^2+...+bn^2))

// 这个公式只比较方向,不考虑长度, 如果一个向量长度 为 2, 另一个为 3, 但方向相同, 相似度仍为 1

```

## 七、计算机真的理解了语义吗？

这里需要注意：Embedding 并不是真的像人一样理解。

它没有：

- 看到苹果想到水果
- 看到股票想到金融

它学习的是：

**词语和词语之间在大量文本中的统计关系。**

人类理解：

- 苹果是一种水果

模型学习：

```
苹果
经常和
水果、吃、甜、香蕉
一起出现
```

最终：这些关系被编码到了向量空间,作为苹果这个词的多维向量的其中一个数字。

## 八、向量知识库的构建及查询流程

1. Embedding 模型的训练：

```
大量文本 (数据准备)
    ↓
学习词语之间的关系
    ↓
调整向量位置
    ↓
形成语义空间
    ↓
构建完整的Embedding模型
```

这里是训练模型的过程，但实际应用中我们通常使用预训练好的 Embedding 模型。

2. 知识库构建：

利用市面上成熟的 Embedding 模型,商用的或开源的都可以。

```

上传文件
    ↓
Chunk 切分
    ↓
每个 Chunk 500 个字符
    ↓
接入向量模型API
    ↓
Tokenization 切分token
    ↓
Token 序列
    ↓
每个 Token 转换为向量表示
    ↓
Transformer 处理 Token 之间的关系
    ↓
Pooling（聚合）
    ↓
得到 Chunk 的最终向量
    ↓
存入本地向量数据库

```

3. 知识库搜索

```
输入文本
    ↓
接入向量模型API
    ↓
Tokenization 切分token
    ↓
Token 序列
    ↓
每个 Token 转换为向量表示
    ↓
Transformer 处理 Token 之间的关系
    ↓
Pooling（聚合）
    ↓
得到输入文本的最终向量
    ↓
本地向量数据库计算向量近似度查询
    ↓
判断语义相似度
    ↓
找到最相似的 Chunk
    ↓
返回相关文本片段
```

这里输出的是一段文字的向量表示，而不是一个词的向量表示。

## 九、总结

计算机虽然不理解文字，但是可以通过向量空间中的数学关系计算，判断内容之间的相似程度。
