# Embedding 的本质：构建语义空间

## 一、先说结论

当我们说"Embedding"(向量化)的时候,本质上是在做一件事情:

> 把文字、图片、代码 等非结构化信息, 转换成一串数字坐标,让计算机能够判断内容之间的语义关系.

简单来说: Embedding 是为了让计算机可以理解内容之间的**相似性**的一种方式

## 为什么需要 Embedding?

在计算机的世界中只有 0 和 1, 它不能直接理解文字、图片等非结构化数据。

eg:

```
我喜欢苹果手机

apple phone is great!
```

这两句话虽然表达的意思相似, 但是对计算机来说,他们就只是两串不同的字符串。

因此, 需要有一种方式:

> 把这些文字转换成可以表达**语义关系**的数字

这就是 Embedding 存在的意义

## Embedding 到底做了什么?

输入一句话:

> "我喜欢苹果手机"

经过 Embedding 模型:

```
我喜欢苹果手机
    ↓
Embedding Model
    ↓
[0.1, 0.9, 0.3, 0.7, ...] // 向量

```

这串数字就是**向量(Vector)**

Embedding 做了这样一件事 :

> 接收一串文字,经过 Embedding 之后, 最终输出了一串**多维数字**, 这一串数字就是向量。

## 向量到底表示了什么?

从数学这个科目来讲,我们都学习过向量, **向量是一个有方向和长度的箭头**, 通常我们接触到的是向量都是二维或者三维的,比如 (2, 3) 或者 (1, 2, 3), 我们可以构建一个二维或者三维的坐标系来表示这些向量。

但是 Embedding 输出的向量通常是 100 维、300 维甚至上千维的, 这样的向量无法用坐标系来表示, 但我们可以抽象理解为它是一个**高维空间中的一个点**。

在 Embedding 中, 这些数字被称为**维度(Dimension)**

很多人会问:
第 1 个数字代表什么?
第 100 个数字代表什么意思?

答案是: **单个数字不表示任何具体含义**

真正重要的是:

> **向量与向量之间的距离**

一个向量代表了一个词的语义,无数个向量构成了一个**语义空间**

## 什么是语义空间

可以把 Embedding 的语义空间 理解为一个**球体的表面**

> 每个向量都对应着这个球面上的一个点, 从圆心出发,到每个点的距离都是一样的,但是方向不同, 两个向量之间的夹角的余弦就代表了语义的相似度.

例如：

```

我想学习 AI
↓
[0.125, -0.231, 0.333, 0.222, 0.111,...]


我想学习人工智能
↓
[0.127, -0.229, 0.331, 0.220, 0.113, ...]

```

虽然两串数字不同，但因为它们表达的意思非常接近，所以生成的向量也会比较接近。

**如何计算两个向量的相似度：**

```
[a1, a2, a3, ... , an] // 向量 1

[b1, b2, b3, ... , bn] // 向量 2

// 计算两个向量的夹角余弦值
相似度  = cosα = (A·B)/(|A|*|B|) = (a1*b1 + a2*b2 + ... + an*bn) / (sqrt(a1^2+a2^2+...+an^2) * sqrt(b1^2+b2^2+...+bn^2))

// 这个公式只比较方向,不考虑长度, 如果一个向量长度 为 2, 另一个为 3, 但方向相同, 相似度仍为 1

```

**两句话的语义越接近,它们的余弦相似度越接近 1, 距离越远, 余弦相似度越接近 0.**

## 整个流程串起来

向量存储:

```
输入文本
↓
Embedding 模型
↓
生成向量
↓
存入向量数据库
```

向量搜索

```
输入文本
↓
Embedding 模型
↓
计算向量距离
↓
找到语义最相近的内容
```

## 一个非常关键的理解:

很多人以为 Embedding 输出的是

- 关键词
- 标签
- 分类

其实都不是, Embedding 输出的是**在语义空间中的一串能代表语义的坐标点数字**, 比较的是不同向量之间的夹角大小。

```
用户提问：

请假需要提前多久？

即使知识库里写的是：

员工请假需要提前一天提交申请。

Embedding 仍然能够判断：

这两个内容表达的是相近的意思。
```

基于这样的能力,支撑起了 RAG 知识库和大模型 等技术的语义理解能力。

## 为什么需要向量搜索

- 传统关键词匹配无法处理语义相似性问题
- 向量搜索可以理解语义相似度
- 通过计算向量夹角，找出最接近的语义内容
- 这样就能实现更智能的问答系统

## Embedding 和 LLM 的关系

基础认知:

- Embedding 是 LLM 的核心组件之一
- 它将文本转换为向量表示
- 使得模型能够理解和处理语义信息
- 为后续的语义搜索和理解提供基础支持

不同点:

1. 传统 Embedding：最终只保留一个（或几个）向量来表示整段文本。
2. LLM：每个 Token 都有自己的向量，而且这些向量会随着上下文不断变化。

## 总结

Embedding 的本质是：

> 把文本转换成向量，并通过向量之间的距离来表示语义相似度。

它不会回答问题，也不会生成文本。
它最核心的作用就是：

> 让计算机能够"理解"哪些内容意思相近，从而找到最相关的信息。
