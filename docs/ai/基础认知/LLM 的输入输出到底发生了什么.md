# LLM 的输入输出到底发生了什么

## 一、先说结论

当你向 LLM 提问的时候，实际发生的事情可以概括为：

> 先 **将文本转换为 Token** -> **模型进行概率计算** -> **循环生成新的 token** -> **再变回文本**

## 二、第一步：输入文本转换 token

你输入一句话： **我想学习 AI**
大模型不会直接处理这句话，而是先转换成 token

> ["我", "想","学", "习"， “AI”]
> 这一步叫做**Tokenization(分词、编码)**

## 三、第二步：模型开始预测（概率计算）

现在模型拿到了第一步输出的 一串 Token，接下来它做的事情是：

> 预测“下一个 token 是什么”
>
> 例如： 我想学习 AI → 可以（预测）
>
> 就会变成：["我", "想","学", "习"， “AI”, "可以"]
>
> 在这一串 token 后添加新的 token，这也是为什么我们看大模型的回答的时候，不停的以一个字或一个词的形式返回的。

## 四、第三步：不断重复生成

模型不会一次性生成整段话，而是：**一步一步的生成**,整体的过程大概如下：

> ["我", "想","学", "习"， “AI”] → ["我", "想","学", "习"， “AI”, "可以"]
>
> ["我", "想","学", "习"， “AI”, "可以"]→ ["我", "想","学", "习"， “AI”, "可以","从"]
>
> ["我", "想","学", "习"， “AI”, "可以","从"]→ ["我", "想","学", "习"， “AI”, "可以","从"，“基础”]
>
> ["我", "想","学", "习"， “AI”, "可以","从"，“基础”]→ ["我", "想","学", "习"， “AI”, "可以","从"，“基础”，“开始”]
>
> ........................

这里每一步都在重复的做一件事: **预测下一个 Token**

## 五、第四步：Token → 文本

当生成完成后,我们得到了这样一串 token

> ["我", "想","学", "习"， “AI”, "可以","从"，“基础”，“开始”]

然后被转换回文本

> 我想学习 AI 可以从基础开始

## 六、 整个流程串起来

```
输入文本
↓
Token化
↓
模型预测（一个一个Token）
↓
生成Token序列
↓
转换回文本
↓
输出给用户
```

## 七、一个非常关键的理解

模型从头到尾都没有：

- 理解句子
- 思考问题
- 组织逻辑

它做的始终是: **基于已有的 token，预测下一个 token**

**那为什么你会感觉它好像理解了？**

- 它训练过海量的人类文本
- 学到了“问题-回答”的模式

所以它看起来像是在：

- 在理解问题
- 在组织答案

但本质依然是：**在生成“最像人类回答的文本”**

## 八、这种生成模式有什么影响？

因为它是逐步的生成,到后面会出现：

- 回答不稳定
- 有时候前后不一致
- 可能“编答案”

说明一下：这不是 bug，这是由大模型的本质决定的：**它是生成模型，不是查询系统**

## 九、总结

LLM 的输入输出过程，本质是： **把文本转换成 token,并逐步的预测生成新的 token，再还原成文本的过程**
