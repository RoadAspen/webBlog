# Token 是什么?为什么不是“字/词”

## 一、先说结论

Token 是 LLM 实际处理的“最小单位”

> LLM 不是在处理“文字”， 而是在处理 Token 序列

## 二、为什么不是字或者词呢？

直觉上我们会觉得：

- 中文 → 按照“字”
- 英文 → 按照“单词”
  但实际上不是的

**举个英文例子：**

> unbelievable
>
> 在模型里实际会被拆解成： ["un", "believ" , "able"]
> 类似于词根词缀的形式

**举个中文例子：**

> 输入： 人工智能
>
> 在模型里实际会被拆解成： ["人工", "智能"]或者是["人", "工", "智能"]

**一个知识点：**

> Token 是 模型自己切出来的， 并不是人定义的词， **不同的大模型解析出来的 token 也不一样**

## 三、为什么要这么做?

**因为模型要处理的是：**

> 各种语言 + 各种表达 + 各种符号

如果只用“词”，就会有以下问题：

- 新词（忽然就不认识了）
- 拼写变化
- 不同的语言结构

所以大模型就采用了一种方式：

> 把文本拆成“可复用的小片段” （token）

## 四、token 的本质到底是什么？

你可以这么理解：

> token 是一种“文本编码单元”

它的作用是：

- 把人类语言 → 转换成 模型可处理的数据
- 作为模型输入输出的基本单位

## 五、一个非常重要的理解

你输入一句话，比如：

> 我想学习 AI
>
> 大模型内部把它切分成 token，可能是： ["我"， “想” ， “学习” ， “AI”]
>
> 模型做的事情是： token -> token -> token
>
> 而不是：句子 -> 理解 -> 回答; 这是人类大脑的处理逻辑

## 六、token 还有其他的意义吗？

token 处理便于大模型计算之外，还有其他的用途

**1. 输入长度限制**

> 模型不是限制"字数"，而是“token 数量”

**2. 成本计算**

> API 也是按照 “token 数量”收费

**3.上下文能力**

> 模型记住的不是有“多少句话”，而是有多少 个“token”

## 七：总结

**token 不是字也不是词，而是模型为了处理语言而定义的“最小文本单位”， LLM 不是在“读语言”，而是在“读 token 序列”**
