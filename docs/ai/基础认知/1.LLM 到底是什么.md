# LLM 到底是什么（从概率预测理解）

## 一、先说结论

LLM (Large 大 Language 语言 Model 模型) 本质上是: **一个"基于上下文, 对下一个 token 进行概率预测的模型"**
你也可以把它理解成一句话:

> 它不是 "理解语言的人", 而是 "目前人类最强的文本续写机器", 在已有文本的基础上, 预测接下来最有可能出现的内容

## 二、LLM 能做什么?

一个直观的例子, 在不联网的情况下, 基于一个训练好的大模型,我们输入一句话:

> "今天的天气很"
>
> 模型并不会理解天气,它做的是:
>
> 根据训练中见过的海量文本.计算每个可能词的概率分布

**模型开始计算: 下一次词是什么?**

| 词   | 概率 |
| ---- | ---- |
| 好   | 0.32 |
| 不错 | 0.25 |
| 糟糕 | 0.18 |
| 热   | 0.15 |

然后: **从中选择一个最可能的词输出**

## 三、为什么它能说完整的话?

**关键在于它在不断的重复这个预测过程**

> 今天的天气很 + 好 + 适合 + 出门 ...
>
> 通过不断重复"预测下一个词",生成完整句子

**但这里有一个重要区分——"续写"和"对话"是两回事:**

上面的例子其实是在演示**预训练 (Pre-training)** 阶段模型获得的能力: 看到上半句 → 续写下半句。本质上就是一个"超级自动补全引擎",你给它一段文本,它接着往下写,并不知道自己是在"回答问题"还是在"扮演角色"。

而我们日常使用 ChatGPT, 豆包 这类产品时,感觉它在"回答你、理解你",这是**后训练 (Post-training)** 的功劳:

- **指令微调 (SFT)** : 用大量"问题-答案"配对数据训练模型,让它学会"用户这段话是在问我问题,我应该给出有用的回答"
- **人类反馈强化学习 (RLHF)** : 让人类对模型回答打分,模型学会什么才是"好的回答"——有帮助、有礼貌、拒绝不当请求

我们熟知的一个人叫姚舜宇,他就是通过"预训练+后训练"的方式,打造了豆包这个产品。

> **简单说: 预训练让它"能说",后训练让它"会说人话"。** 前者学会的是"下一个词最可能是什么",后者学会的是"作为助手,我现在应该怎么说"。

## 四、一个关键的误解

很多人以为的 AI:

1. LLM = 会思考的 AI
2. LLM = 有知识的系统
3. LLM = 类人智能

但实际上的 AI:

1. 海量文本学习 + 统计规律 + 概率预测
2. 它不是在"思考", 而是在"生成看起来像思考的文本"
   LLM 做的事情始终只有一个:

> 在当前的上下文信息下, 预测下一个最合理(根据概率)出现的词(Token)

**海量文本学习**: 学习的是大量人类知识的结论.

## 五、总结

**LLM 不是一个会思考的系统, 而是一个非常强大的"文本预测模型"**
