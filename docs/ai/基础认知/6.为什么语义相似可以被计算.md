# 为什么“语义相似”可以被计算

上一章我们讲了, Embedding 是如何将文字转换成向量，并把文本映射到一个高维的语义空间。。
在这个空间中

- 相似的语义内容距离更近。
- 不同的语义内容距离更远。

那么, 一个新的问题:

> 为什么一串数字之间的距离，可以代表两个文本的语义相似程度？

## 一、先说结论

语义相似度之所以可以被计算

> **因为 Embedding 模型在训练过程中，学习了大量文本之间的关系,并将这种关系编码到了向量空间中**

简单来说：

> 模型不是人为规定“苹果”和“香蕉”应该靠近，而是在大量文本中学习出了它们之间的关系。

## 我们需要达成一个共识

市面上可用的 Embedding 模型也是经过大量的数据训练出来的,所以它能学习到语义关系。

1. 如果 Embedding 模型没有经过任何的训练,那么它就无法表达语义关系。输出的就是是一个完全随机的向量,而不是有意义的语义向量。

2. 在 Embedding 训练的开始,模型会对一些文字进行随机初始化,当训练的过程中,它会根据输入的文本的关联度,不断的调整这些向量,直到这些向量能够准确表达语义关系为止。

3. 当我们使用一个训练好的 Embedding 模型,我们得到的向量是具有语义含义的。

接下来我们说的都是基于训练好的 Embedding 模型。

## 二、Embedding 并不是打标签

很多人认为: 苹果 -> [0.12, -0.35, 0.67, ...]

- 第一个数字代表水果
- 第二个代表颜色

不是的, 我们有理由认为某个数字确实是代表了某种语义，但不是像上面那样简单粗暴地认为第一位是水果,第二位是颜色,而是通过复杂的数学运算来表达语义关系的。

Embedding 并不是给每个词打标签。
它做的是:

> 根据大量的文本中的使用规律,学习每个词,每句话之间的语义关系,确定它们在语义空间中的位置

## 三、模型是如何学习语义关系的？

假设模型在训练过程中看到三段文字：

1. 我喜欢吃苹果
2. 我喜欢吃香蕉
3. 我喜欢吃西瓜

模型就会发现:有一些词经常出现在类似的位置:

> 喜欢吃\_\_

于是模型就逐渐学习:

苹果、香蕉、西瓜 它们在同一个语境下出现，所以它们是具有某种相似的语义关系的。

**因此: 它们在语义空间中会彼此靠近。**

再例如：
模型看到：

- 苹果发布了新手机
- 苹果推出了新的电脑

这里的"苹果"和：

手机
电脑
芯片
科技公司

一起出现。

所以模型会学习到：

这里的苹果代表的是：Apple 公司,而不是水果。

> 这个过程的外在表现就是: 代表苹果这个词的向量中的数字在不停地调整。

如果把训练过程用二维简化来可视化，大致是这样的——

注意：**多维空间无法直接观察，但可以通过降维来理解。** 把 Embedding 训练理解成"不断调整地图上的位置"（二维化展示，便于理解）：

```
初始状态（随机分布）：

苹果       ●

香蕉                 ●

手机                       ●
```

模型不知道它们之间的关系。随着不断训练，模型发现苹果和香蕉经常出现在相似的环境（都是食物语境），于是拉近它们：

```
第一轮调整后：

苹果       ●

香蕉        ●

水果          ●
```

同时，模型发现手机、电脑和科技公司经常出现在相似的环境，于是把它们聚到另一个区域：

```
最终形成的语义空间（二维简化）：

          水果区域
       苹果 ●
            ● 香蕉
                         科技区域

                      手机 ●
                         ●电脑

```

> 这就是语义空间的形成过程：从随机分布 → 相似语义聚拢 → 不同语义分开。

## 五、为什么距离代表了语义相似度？

从第三、四节的过程可以看出，语义空间不是一开始就有序的——

1. **训练前**：所有向量的位置是随机的，距离没有语义含义
2. **训练中**：模型不断发现"苹果"和"香蕉"频繁出现在相似语境，于是一轮轮把它们拉近；同时发现"手机"和"芯片"总在一起出现，于是聚出另一个簇
3. **训练后**：位置变得有序，向量之间的距离开始代表语义关系

关键洞察：

> **距离不是被人为定义的，而是从海量文本的统计规律中自然涌现的。**

换句话说，Embedding 模型并没有"学会苹果和香蕉都是水果这条知识"。它只是观察到这两个词总是在相似的上下文（喜欢吃**、**很甜、买\_\_）里出现，于是把它们放在了向量空间的相近位置。

**语义关系是统计规律的外在表现，不是模型真的理解了语义。** 这一点在第八节会进一步讨论。

## 六、如何计算两个向量的相似度

在模型中使用两个向量的**夹角余弦值**来衡量它们的相似度。

```
[a1, a2, a3, ... , an] // 向量 1

[b1, b2, b3, ... , bn] // 向量 2

// 计算两个向量的夹角余弦值公式
相似度  = cosα = (A·B)/(|A|*|B|) = (a1*b1 + a2*b2 + ... + an*bn) / (sqrt(a1^2+a2^2+...+an^2) * sqrt(b1^2+b2^2+...+bn^2))

// 这个公式只比较方向,不考虑长度, 如果一个向量长度 为 2, 另一个为 3, 但方向相同, 相似度仍为 1

```

## 七、为什么用余弦而不是欧氏距离

我们知道向量之间有多种相似度计算方法，比如：欧氏距离和余弦相似度。

- **欧氏距离** 比较的是两个向量之间的实际距离
- **余弦相似度** 比较的是两个向量的方向。

但是对于 Embedding 来说：

> **方向比长度更重要。**

假设有两个向量：

```
A = (1,1)

B = (2,2)
```

### 欧式距离计算什么?

从直角坐标轴上看, A 和 B 的位置关系如下：

```
       B ●
        /
       /
      /
     ● A
    /
   /
  /
O/_____
```

欧氏距离 就是两点之间的直线距离，也就是三角形的斜边长。

> 从数学上讲，欧氏距离是：√((2-1)²+(2-1)²) = √2 ≈ 1.414

但是，这两个向量的方向是一样的

### 余弦相似度计算什么?

余弦相似度不关心长度, 只关心**两个向量是不是朝着同一个方向**
比如:

```
A = (1,1)
B=(100,100)
```

长度相差 100 倍, 但是方向完全一致,余弦相似度仍然为 1,方向完全一致

### 总结

- **欧氏距离**衡量的是两个向量的**绝对距离**，即长度差异
- **余弦相似度**衡量的是两个向量的**相对方向**，即角度差异

**当归一化后, 欧式距离和余弦相似度是等价的**

## 八、计算机真的理解了语义吗？

这里需要注意：Embedding 并不是真的像人一样理解。

它没有：

- 看到苹果想到水果
- 看到股票想到金融

它学习的是：

**词语和词语之间在大量文本中的统计关系。**

人类理解：

- 苹果是一种水果

模型学习：

```
苹果
经常和
水果、吃、甜、香蕉
一起出现
```

最终：这些关系被编码到了向量空间,作为苹果这个词的多维向量的其中一个数字。

## 九、向量知识库的构建及查询流程

1. Embedding 模型的训练：

```
大量文本 (数据准备)
    ↓
学习词语之间的关系
    ↓
调整向量位置
    ↓
形成语义空间
    ↓
构建完整的Embedding模型
```

这里是训练模型的过程，但实际应用中我们通常使用预训练好的 Embedding 模型。

2. 知识库构建：

利用市面上成熟的 Embedding 模型,商用的或开源的都可以。

```

上传文件
    ↓
Chunk 切分
    ↓
每个 Chunk 500 个字符
    ↓
接入向量模型API
    ↓
Tokenization 切分token
    ↓
Token 序列
    ↓
每个 Token 转换为向量表示
    ↓
Transformer 处理 Token 之间的关系
    ↓
Pooling（聚合）
    ↓
得到 Chunk 的最终向量
    ↓
存入本地向量数据库

```

3. 知识库搜索

```
输入文本
    ↓
接入向量模型API
    ↓
Tokenization 切分token
    ↓
Token 序列
    ↓
每个 Token 转换为向量表示
    ↓
Transformer 处理 Token 之间的关系
    ↓
Pooling（聚合）
    ↓
得到输入文本的最终向量
    ↓
本地向量数据库计算向量近似度查询
    ↓
判断语义相似度
    ↓
找到最相似的 Chunk
    ↓
返回相关文本片段
```

这里输出的是一段文字的向量表示，而不是一个词的向量表示。

> **实际落地**：本文描述的整套原理（分布式语义学习 → 向量生成 → 余弦相似度检索），正是 GLM embedding-3、text-embedding-3 等商用 Embedding 模型的底层工作机制。理解这些原理，有助于在实际项目中判断"为什么某个查询精准命中、而另一个查询却漏掉了"——关于技术选型与实践，详见第五篇：[Embedding 的本质](<5. Embedding 的本质.md>)。

## 十、总结

计算机虽然不理解文字，但是可以通过向量空间中的数学关系计算，判断内容之间的相似程度。
